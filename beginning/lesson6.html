Lesson 6

Back and Forth

← Back to Home

In this lesson, you'll learn how to make shapes do something and then undo it. For example, you'll make shapes move to the right and then reverse direction when they touch an edge. You'll see shapes slowly change color and then return to the original color. You'll see other shapes that get bigger and then smaller.

You'll also learn how the background command and variables help to create these back-and-forth changes.

1. Many Ellipses Sweeping

Tinker

Click ▶Play. You'll see many ellipses that appear one after the other, from left to right.
Try these challenges: Change the script to make the ellipses appear further apart. Change the script to make the ellipses move from right to left.

To make the ellipses appear further apart, find this line of code:
x = x + 8;
Change 8 to a larger number and then click ▶Play.

Why does this work? The command
x = x + 8;
increases x by 8 over and over each time function draw() repeats. So x becomes 0, 8, 16, and so on. At the same time, the command
ellipse(x, 100, 6, 6);
draws ellipses with their sideways positions at whatever number x got. That means that the ellipses appear at 0, 8, 16, and so on, with each ellipse a little further to the right, 8 pixels apart.

If we add a larger number, like x = x + 12; instead of x = x + 8; then the ellipses will appear at 0, 12, 24, and so on — spreading them further apart.

To make the ellipses move from right to left, you must make two changes in the script. Here is one possible solution: 

let x = 250;
...
function draw() {
  ellipse(x, 100, 6, 6);
  x = x - 8;
}

On line 1, we give x a number like 250, which places the ellipse near the right edge of the canvas. And later, we subtract a small number like 8 from x each time function draw() runs. This causes each new ellipse to appear a little more to the left than its predecessors.

2. One Ellipse

Tinker

Click ▶Play. You'll see one ellipse traveling across the canvas from left to right. The code is almost the same as in the previous step, but there's one difference. Can you find it?
Try these challenges: Delete the background command. What happens? Why? Click Revert to restore the original script. Now cut-and-paste the background command into function setup() instead of function draw(). Then click ▶Play. What happens? Why?

If the background command is in function draw(), you see only one ellipse. But if you delete the background command OR if background is in function setup(), you see many ellipses. Here's why.

If the background command is in function draw(), then the background color is painted on the canvas over and over, about 60 times every second. Each time it is painted, the background covers everything, including previously-drawn shapes like the ellipse.

When you delete the background command, there is no background! The ellipses painted by function draw() — which runs over and over — all appear on the canvas.

When the background command is in function setup(), it is painted just once when the project begins. The ellipses painted by function draw() — which runs over and over — all appear on the canvas after the background is drawn.

3. Stop the Motion

Tinker

Click ▶Play. You'll see one ellipse that stops just before the right edge. There's also a new command on line 11 called noLoop. Let's investigate.
Try these challenges: Change the script to make the ellipse stop halfway across. Change the script to make the ellipse move from right to left, stopping halfway across.

To make the ellipse stop halfway across, we change line 11 to:
if (x > 125) noLoop();
This command is read: "if x becomes greater than 125, stop looping." By stop looping we mean that function draw() should stop repeating. This stops the animation. The symbol > means "greater than." We check whether x > 125 because x controls the sideways position of the ellipse and 125 is exactly halfway across the canvas.

Here's one way to make the ellipse move from right to left and stop halfway across.
Change line 1 to:
let x = 250;
Since x controls the sideways position of the ellipse, this change places the ellipse at the right side of the canvas.
 Change line 11 to:
x = x - 8;
We subtract 8 or any other small number from x so that each ellipse appears a little to the left of previous ellipses.
 Change line 13 to:
if (x < 125) noLoop();
We check whether x is less than 125, halfway across the canvas.

5. Bounce

Tinker

Click ▶Play. You'll see an ellipse that moves from left to right, but this time when it reaches the right edge it reverses direction as if bouncing. Why?
Try these challenges: Change line 2 so that step gets a bigger number. Then click ▶Play. What happens? Change line 2 again so that step gets 8. Then click ▶Play. When step is 8, the ellipse slows down after it touches the right edge. Why? Fix the script so that the ellipse bounces and maintains the faster speed. Change the script so that the ellipse bounces when it touches the left edge, too. Hint! You'll need to add another if command. üôÇ

To make the step variable get 8, change line 2 to:
let step = 8;

What is step? On line 13, we add step to x, over and over. That means each ellipse will appear step pixels further to the right than previous ellipses. The bigger the step is, the further apart the ellipses appear. And the futher they are apart, the faster they appear to travel. So step controls the ellipse speed.

To make the ellipse bounce and then travel at the same faster speed, change line 14 to:
if (x > 250) step = -8;
The number -8 makes the ellipse travel from right to left (instead of left to right). If the ellipse starts at speed 8, then -8 makes it travel in the opposite direction at the same speed.

To make the ellipse bounce when it touches the left edge, too, add the following command at the end of the script:
if (x < 0) step = 8;
This command reads, "If x is less than 0, then reset the step variable to 8."
The variable x becomes less than 0 when the ellipse reaches the left edge. At that moment, we want the ellipse to begin traveling to the right again. We can make that happen if we change the step variable back to a positive number, like 8.

6. Bounce Everywhere

Tinker

Click ▶Play. You'll see an ellipse bounce off the bottom and then move through the right edge and disappear. We need to make the ellipse bounce off all edges.
Try these challenges: Add a command that makes the ellipse bounce off the right edge. Add another command that makes the ellipse bounce off the top edge. Add one last command that makes the ellipse bounce off the left edge. Change the script so that the ellipse starts moving at a different angle. For example, make the ellipse move towards the right wall instead of towards the bottom.

In this script, the variable x controls the sideways position of the ellipse, and the variable y controls its vertical position.
ellipse(x, y, 20, 20);

The variable xstep controls the sideways speed of the ellipse because it's added to x over and over, each time function draw() runs:
x = x + xstep;
The variable ystep controls the vertical speed of the ellipse because it's added to y over and over, each time function draw() runs:
y = y + ystep;

To make the ellipse bounce off the right edge, add the following command. Just to keep your script organized, add it on line 17.
if (x > 250) xstep = -2;
This command reads, "If x is greater than 250, then reset the xstep variable to -2."
The variable x becomes greater than 250 when the ellipse reaches the right edge. At that moment, we want the ellipse to begin traveling to the left. We can make that happen if we change the xstep variable to a negative number, like -2.

To make the ellipse bounce off the top edge, add the following command. Just to keep your script organized, add it towards the end of the script.
if (y < 0) ystep = 3;
This command reads, "If y is less than 0, then reset the ystep variable to 3."
The variable y becomes less than 0 when the ellipse reaches the top edge. At that moment, we want the ellipse to begin traveling downwards. We can make that happen if we change the ystep variable to a positive number, like 3.

To make the ellipse bounce off the left edge, add the following command. Just to keep your script organized, add it after line 17.
if (x < 0) xstep = 2;
This command reads, "If x is less than 0, then reset the xstep variable to 2."
The variable x becomes less than 0 when the ellipse reaches the left edge. At that moment, we want the ellipse to begin traveling to the right. We can make that happen if we change the xstep variable to a positive number, like 2.

To make the ellipse start at a different angle, change the values given to xstep and ystep at the beginning of the script. Here's one way to change the angle:
let xstep = 4;
let ystep = 2;
 These changes make the ellipse travel faster to the right than downwards, which changes its traveling angle.

7. Fade In and Out

Tinker

Click ▶Play. You'll see a rectangle that fades out and fades back in. Let's investigate.
Try these challenges: Notice that the fill command contains a fourth argument which adds transparency. In this case, the transparency is a variable that's changing. Can you make it change more slowly? More quickly? Click Revert to restore the original script. Add a smaller rectangle in a different color. Make it fade in and out, too.

In this script, the variable t controls the transparency of the ellipse.
fill(325, 100, 100, t);

The variable tstep controls how fast the transparency changes because it's added to t over and over, each time function draw() runs:
t = t + tstep;

If tstep is positive, then the transparency t gets bigger, and the background fades in. If tstep is negative, then the transparency t gets smaller, and the background fades out.

Remember that when transparency t reaches 1, the background is completely faded in, and you see its full color. When transparency t goes back down to 0, the background is completely faded out, and you see black.

To make the background fade out more quickly, give tstep a number that's more negative, like -0.03 instead of -0.01. You'll need to make 2 changes.
On line 1:
let tstep = -0.03; On the last line:
if (t > 1) tstep = -0.03;
This line reads, "If t becomes greater than 1, reset tstep to -0.03." This makes the background fade out after it has faded in fully.

When the script begins, the background fades out. After it fades out completely, it starts fading in again on line 19:
if (t < 0) tstep = 0.01;
This line is read, "If t becomes less than 0 (and the background is completely faded out), reset tstep to 0.01." Since tstep is positive, the background fades in.

To make the background fade in more quickly, change line 19 so tstep gets a larger, positive number like this:
if (t < 0) tstep = 0.03;

Here's one possible way to create a second rectangle. Add these lines of code:
fill(195, 100, 100, t);
rect(50, 50, 150, 100);

8. Fade Out and In

Tinker

Click ▶Play. You'll see two rectangles. One fades out, and the other fades in.
Try these challenges: Change the code so that both rectangles fade in. Click Revert to restore the original code. The transparency of the second rectangle is 1 - t. How does that make the second rectangle fade in the opposite direction from the first rectangle?

In this script, the variable t controls the transparency of the ellipse.
fill(325, 100, 100, t);

To make both rectangles fade in, on line 15 change the transparency for the second rectangle from 1 - t to t:
fill(195, 100, 100, t);

If one rectangle has transparency t and the other one has transparency 1 - t, the two transparencies move in opposite directions. Let's explore why.

If t is 0.01 then 1 - t is 0.99.
If t is 0.02 then 1 - t is 0.98.
If t is 0.03 then 1 - t is 0.97.
So while t gets more positive, 1 - t gets less positive. That means the first rectangle fades in and the second rectangle fades out.

If t is -0.01 then 1 - t is
1 - -0.01
= 1 + 0.01
= 0.01.

If t is -0.02 then 1 - t is 0.02.
If t is -0.03 then 1 - t is 0.03.
So while t gets more negative, 1 - t gets more positive. That means the first rectangle fades out and the second rectangle fades in.

9. Grow and Shrink

Tinker

Click ▶Play. You will see an ellipse. We need to make it grow and shrink.
Try these challenges: Add a command that makes the ellipse grow. Hint! The command will include the variables size and sizestep. Add another command to make the ellipse shrink when it gets too big. You can decide what's too big!

To make the ellipse grow, add this command at the end of the script:

To make the ellipse shrink when it reaches a certain size, like 100 pixels, add a command like this at the end of the script:

You don't have to use 100 as the maximum size. Try to find a maximum size you like!

Example #1 — Bouncing Circles

For this example, see the first GIF image at the beginning of this lesson. The yellow circle moves straight across the canvas and back again. The other two circles move diagonally. The two diagonally-moving circles reflect each other's movement, like the shapes in the Reflections lesson. Feel free to make the same animation or something very different! The code below includes all the variables you'll need, although you may wish to add or change them; many different strategies work successfully! The code contains just one ellipse, so you may want to add more. The ellipse has no fill and a big stroke weight that makes a thick outline. This outline is partially transparent because of the fourth argument in the stroke command, 0.2. 

/* *** BOUNCING CIRCLES *** */

let x = 0;
let xstep = 1;
let y = 0;
let ystep = 2;

function setup() {
  createCanvas(400, 400);
  colorMode(HSB);
}

function draw() {
  background(0, 0, 100);

  noFill();
  strokeWeight(50);

  stroke(105, 100, 100, 0.8);
  ellipse(x, y, 200, 200);


}

Example #2 — Grow and Shrink

For this example, see the third GIF image at the beginning of this lesson. A large ellipse at the top shows its lower half on the canvas (the other half is there but can't be seen above the canvas). There's a similar large ellipse at the bottom, and they grow and shrink together. Three smaller ellipses at the center of the canvas grow and shrink more quickly. Feel free to make the same animation or something very different! The code below includes only some of the variables you'll need, so you'll need to add more. The example also includes only the two large ellipses, so you'll need to add the smaller ones. 

/* *** GROW AND SHRINK *** */

let sz = 120;
let szstep = 1;

let running = false;

function setup() {
  createCanvas(400, 400);
  colorMode(HSB);
}

function draw() {
  background(30, 10, 30);
  strokeWeight(24);

  noFill();
  stroke(105, 100, 100);
  ellipse(200, 0, sz, sz);
  ellipse(200, 400, sz, sz);


}
