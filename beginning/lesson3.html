createCanvas defines the width and height of the canvas, the drawing area that you see after you click ▶Play. The width and height are measured in pixels. A pixel is the smallest dot that the screen can show. Each pixel is about 1/100 inch wide and tall. This whole canvas is 250 pixels wide and 200 pixels tall.

background changes the color of the canvas.

The numbers in parentheses are called arguments, and they change how commands work. For example, background(0) creates black and background(255) creates white. Arguments between 0 and 255 produce gray, with lighter grays as arguments get bigger.

3. Color

Let's add color to our drawing!

Tinker

Click ▶Play. You'll see a yellow background. It's created by the commands colorMode(HSB) and background(51, 100, 100).
Follow these steps to investigate colors and add your own! Go to HSB Color Picker in a new tab. You'll see 3 sliders. Move them to change colors. Find a color you like! Each slider is represented by a number. Type the numbers one at a time, in order, into the background command below. Make sure there are commas between each number. Don't include ° and % symbols. Remember to click ▶Play. 🙂 You can learn more about these ideas by clicking Color Help at the right.

At HSB Color Picker, you'll see 3 numbers labeled Hue, Saturation, and Brightness. These 3 qualities, taken together, create a color.

Hue is the basic color — red, gold, turquoise, and so on. It is represented as degrees around a color circle. For example, our yellow is at 51°. Since there are 360 degrees in a circle, there are 360 possible hues.

Saturation is the intensity of the color. It is represented as a percentage, with 100% meaning the greatest intensity and 0% meaning the least intensity (which creates white).

Brightness is the shade or amount of black mixed into color. It is represented as a percentage, with 100% meaning no black and 0% meaning all black.

Here' are 3 common background colors, as examples:
background(0, 100, 100); ← red
background(0, 0, 100); ← white
background(0, 100, 0); ← black

In the colorMode command, the letters HSB must be capitalized.

In ellipse(150, 100, 50, 35), the first number, 150, is the ellipse's distance from the left edge. The second number, 100, is its distance from the top.

Here's a picture of the ellipse and distances.
These distances are so helpful for placing images on screen that we give them a special name: coordinates. A shape's distance from the left edge is called its x-coordinate. Its distance from the top is called its y-coordinate.
We write the coordinates like this:
150, 100
The x-coordinate is always first, followed by the y-coordinate. The two coordinates are separated by a comma.

fill(5, 100, 100) makes the inside of the ellipse red.

stroke(0, 100, 0) makes the border around the ellipse black.

strokeWeight(6) changes the thickness of the border, measured in pixels.

The computer runs commands in order, from top to bottom. That means you must place the fill, stroke, and strokeWeight commands before the shape whose color you're changing. Otherwise the colors won't work at all!

6. Random Gray Backgrounds

Tinker

Click ▶Play, several times. Each time you click ▶Play, the background changes to a different shade of gray.
Try these challenges: Change 225 to 100. Click ▶Play several times. What happens? Why? To learn more about these ideas, click Random Gray Backgrounds Help at the right.

When we tuck random(225) into background, the punctuation gets a little tricky! We end up with four parentheses, like this:

The computer always reads the innermost parentheses first. So the computer chooses a random number from 0-225. Then it gives this random number to the background command, producing a random gray color.

It's helpful to understand that the random numbers may not seem to change every time you click ▶Play. That's because it's possible for the computer to pick the same random number twice in a row, just like you can get "heads" or "tails" twice in a row when you randomly flip coins.

Suppose you use a smaller number than 225 in the random command, like this:
background( random(100) )
Then the computer will choose a random number between 0-100, and you'll see random grays that are darker. Remember that lower numbers produce darker grays.

When background contains a single number, you see a shade of gray (including black and white). But when background contains 3 numbers — representing hue, saturation, and brightness — you see colors.

7. Random Color

Tinker

Click ▶Play, several times. Each time you click, the background changes color randomly. The random colors come from the command:
background( random(360), 100, 100 )
which randomly changes hue. (If you can't see the whole background command in the code below, click ■ Stop to show the full code window.)
Try these challenges: Make the fill random. Make the stroke random. Make the strokeWeight random. And most fun of all, change the ellipse position and size randomly!

Notice that in
background( random(360), 100, 100 );
we chose a random number only for the first number, the hue (the basic color). We didn't randomize saturation or brightness. For the random hue, we chose randomly among the 360 possibilities.

But you could randomize saturation and brightness, too. :-) You could write a command like this:
background( random(360), random(100), random(100) );

There are 100 possible saturations and 100 possible brightness values, and we randomly choose one of them.

You can randomize stroke and fill as well. Make any changes that create colors you like!

The argument to strokeWeight can also be random. See if you can figure that out yourself!

8. setup() and draw()

Tinker

Click ▶Play. You'll see dozens of ellipses! Where did they come from?
Notice that the script has been divided into two sections called function setup() and function draw(). These two sections change the order in which the commands run. Here's how it works.
When you click ▶Play, p5.js looks for function setup() and runs any commands it finds there once. Then it looks for function draw() and runs any commands it finds there over and over, endlessly — in this case creating infinite ellipses at random positions. Magic!
Try these challenges: Change the ellipse command to draw ellipses with random sizes. Draw randomly-colored ellipses by adding fill, stroke, and strokeWeight commands in function draw(). Be sure to add them before the ellipse command.

A function is a set of commands that work together to do a specific task. function setup() sets up the drawing by running commands once before the drawing appears. Then function draw() draws the sketch over and over, endlessly.

As you write code, it's helpful to know rules about syntax. Syntax, pronounced SIN-tax, means the arrangement of letters and symbols in commands. If you use correct syntax, the computer will understand what you want to do. If you use incorrect syntax, the computer will show an error message and stop drawing.

Here are some important syntax rules:

  • Most commands end with ;
    which is called a semicolon.
  • You can include extra space before or after commands or numbers. For example, both of the following commands have acceptable syntax:
    background ( 51, 100, 100 );
      background(51,100,100);
  • You can include extra blank lines between commands, usually added for readability.
  • There can be as many commands in function setup() and function draw() as you like, but they all must be written between { and }. You can think of { } like bookends, with all the commands lined up between them like books.
