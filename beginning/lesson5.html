1. Many Rectangles Falling

Tinker

Click ▶Play. You'll see many rectangles that appear one after the other, from top to bottom.
Try these challenges: Look at line 1 of the code:
let z = 0;
This code creates a variable named z that stands in for the number 0.
 In line 1 of the code, change 0 to different numbers between 10 and 100, clicking ▶Play after each change to see the effect. What happens? You've probably discovered that the numbers you type in line 1 change the vertical position where the rectangle starts moving downwards. But line 1, by itself, just associates a number with a variable named z. The code that changes the rectangle's position appears later in the script. Can you find it? Each rectangle appears 8 pixels further down the canvas than the previous rectangle. Can you find the line of code that makes that happen? Change that line to spread the rectangles further apart.

As we mentioned previously, the command let z = 0; creates a variable with the name z that stands in for the number 0. We read this line:
"let z get 0" or "let z become 0"

Inside your computer, the variable z is a location in memory that stores the number 0. When you use variable z in your code, your computer goes to that location in memory, gets the 0 stored there, and then uses that value to run your code.

When you change 0 in line 1 to a different number, the vertical position of the rectangle changes. The rectangle's position on canvas is determined by line 10, later in the script:
rect(125, z, 6, 6);
This code sets the rectangle's vertical position to whatever number z stands in for. So if line 1 contains
let z = 100;
then the rectangle appears at vertical position 100, halfway down the canvas.

The command z = z + 8; controls how far apart the rectangles appear. This code may seem nonsensical: how can z equal z + 8? It doesn't! Your computer understands this line in an entirely different way than in mathematics. When your computer finds a line of code containing a single = it starts at the righthand side and reads the line moving left. In this case, it gets the number 8. Then it finds +, so it knows it's about to add 8 to something. Then it finds z, so it adds 8 to whatever number z is standing in for, and it makes that sum the new value of z.

For example, in our script z starts at 0. So your computer adds 8 + 0 and then updates z to be 8. Since function draw() happens over and over, your computer soon comes to the same line of code again. This time it adds 8 + 8 and updates z to be 16. And so on! As z becomes larger and larger, the rectangles move down the canvas.

The command if (y > 200) y = 0; means "if the variable y becomes greater than 200, reset the variable y to 0."

We read this line:
"if y is greater than 200 then y becomes (or gets) 0."

What does this line do on screen? Remember that vertical position 200 is at the bottom of the canvas, and vertical position 0 is at the top. So as soon as the rectangle reaches the bottom of the canvas, it jumps back to the top and starts falling back down again.

If you change this line of code to something like:
if (y > 44) y = 0;
then as soon as the rectangle reaches the vertical position 44, about 1/5 of the way down the canvas, it jumps back to the top.

If you change this line of code to something like:
if (y > 200) y = 125;
then as soon as the rectangle reaches the bottom of the canvas, it jumps back to vertical position 125, a little more than halfway down the canvas. This makes the ripples appear a little more than halfway down the stack.

On line 1 of the script, the variable h gets 50. That makes the first rectangles yellow.

But on line 16, the variable h changes again. This time, if the variable y becomes greater than 200 then h gets 325. What does that mean? We know that the variable y becomes 200 when the rectangle reaches the bottom of the canvas. At that moment, h becomes 325 and that makes the rectangles turn pink.

If you replace 325 with a different number, then after the rectangles reach the bottom of the canvas they will change to that color.

If you reverse the order of the final lines of code:
if (y > 200) y = 0;
if (y > 200) h = 325;
then the color will never change! Let's see why.

First, your computer checks whether the variable y is greater than 200. If so, then y gets 0. Then your computer reads the next line of code and checks again to see if y is greater than 200. But it isn't! The variable y is now 0, so your computer doesn't complete that next line of code, never changes the variable h, and therefore never changes the color.

11. Diagonals

Tinker

Click ▶Play. You'll see many stacks of ellipses at random sideways positions, and they make diagonals that tilt to the right as they move up the canvas.
Try this challenge: Change the code so that the ellipses make diagonals that tilt left as they move up the canvas.

To make the diagonals tilt left, change line 15 to:
x = x - 1;
Let's see why that works.

When the script begins, the variable x gets 125.
let x = 125;

The variable x controls the sideways position of the ellipse:
ellipse(x, y, 20, 20);
Remember that the sideways positon of the ellipse is called its x-coordinate.

When you click ▶Play the first ellipse appears at sideways position, or x-coordinate, 125. That's exactly halfway across the canvas.

But then x gets a different value:
x = x + 1;
Remember that when the computer encounters a single = it reads the commands from right to left. In this case, the computer gets 1 and then adds it to whatever number x got. So when the script begins, the computer adds 1 + 125, gets 126, and then changes x to 126.

But changing x to 126 also means that the next ellipse will appear at sideways position or x-coordinate 126. That's 1 pixel to the right of the previous ellipse.

And the next ellipse after that is drawn 1 more pixel to the right, at sideways position or x-coordinate 127. The rightward movement of each ellipse makes a diagonal.

To make the diagonals tilt left, we need to subtract 1 each time a new ellipse is drawn. To say this another way, we want to draw the first ellipse at sideways position or x-coordinate 125, and the next ellipse at 124, and the next one at 123, and so on. We want, in other words:
x = x - 1;

/* *** FALLING RECTANGLES *** */

let h = 50;
let y = 0;

function setup() {
  createCanvas(250, 200);
  colorMode(HSB);
  background(30, 10, 30);
  frameRate(10);
}

function draw() {
  fill(h, 100, 100);
  rect(4, y, 20, 20);
  rect(34, y, 20, 20);
  y = y + 8;

  if (y > 200) h = 325;
  if (y > 200) y = 0;
}

/* *** RISING ELLIPSES *** */

let x = 125;
let y = 200;

function setup() {
  createCanvas(250, 200);
  colorMode(HSB);
  background(30, 10, 30);
}

function draw() {
  fill(0, 100, 100);
  ellipse(x, y, 20, 20);
  y = y - 12;

  if (y < 0) x = random(200);
  if (y < 0) y = 200;
}
