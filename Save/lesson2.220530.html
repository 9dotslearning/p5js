createCanvas defines the width and height of the canvas, the drawing area that you see after you click ▶Play. Its size is measured in pixels. A pixel is the smallest dot that the screen can show; it's about 1/100 inch wide and tall. This canvas is 250 pixels wide and 200 pixels tall.

background changes the color of the canvas.

The numbers in parentheses are called arguments, and they change how commands work. For example, background(0) creates black and background(255) creates white. Arguments between 0 and 255 produce gray, with lighter grays as arguments get bigger.

At HSB Color Picker, notice the 3 numbers labeled Hue, Saturation, and Brightness. These 3 qualities, taken together, create a color.

Hue is the basic color — red, gold, turquoise, and so on. It is represented as degrees around a color circle. For example, our yellow is at 51°.

Saturation is the intensity of the color. It is represented as a percentage, with 100% meaning full color and 0% meaning no color (white).

Brightness is the shade or amount of black mixed into color. It is represented as a percentage, with 100% very bright (no black) and 0% meaning the color has become completely black.

6. Random Gray Backgrounds

Tinker

Click ▶Play, several times. The ellipse and the color are both gone (for a moment). Each time you click ▶Play, the background changes to a different shade of gray.
Try thess challenges: Change 255 to a smaller number. What happens? Make the random background command look like this:
background( random( 127, 255 ) );
You'll see only light grays. Why? To learn more about these ideas, click Random Gray Backgrounds Help at the right.

When we tuck random(255) into background, the punctuation gets a little tricky! We end up with four parentheses, like this:

background( random(255) );

The computer always reads the innermost parentheses first. So the computer chooses a random number from 0-255. Then it gives this random number to the background command, producing a random gray color.

It's helpful to understand that the random numbers may not always change when you click ▶Play. That's because it's possible for the computer to pick the same random number twice, just like when you flip coins you can get "heads" twice in a row.

Suppose you use a smaller number than 255 in random, like this:
background( random(100) )
Then you'll get darker grays! Remember that lower numbers — random or not — produce darker grays.

The command random( 127, 255 ) chooses a random number between 127 and 255. So the command background( random(127, 255) ) creates lighter grays.

which randomly changes hue. (If you can't see the whole background command, click ■ Stop to show the full code window.)
Try these challenges: Make the fill random. Make the stroke random. Make the strokeWeight random. And most fun of all, change the ellipse position and size randomly!

Notice that in
background( random(255), 100, 100 );
we chose a random number only for the first number, the hue (the basic color). We didn't randomize saturation or brightness.

But you could randomize saturation and brightness, too. :-) You could write a command like this:
background( random(255), random(100), random(100) );
You can make similar changes to stroke and fill as well. Make any changes that create colors you like!

8. setup() and draw()

Tinker

Click ▶Play. You'll see dozens of ellipses! Where did they come from?
Notice that the script has been divided into two sections called function setup() and function draw(). These two sections to change the order in which the commands run. Here's how it works.
When you click ▶Play, p5.js looks for function setup() and runs any commands it finds there once. Then it looks for function draw() and runs any commands it finds there over and over, endlessly — in this case creating infinite ellipses at random positions. Magic!
Try these challenges: Change the ellipse command to draw ellipses with random sizes. Draw randomly-colored ellipses by adding fill, stroke, and strokeWeight commands in function draw(). Be sure to add them above the ellipse command.

A function is a set of commands that work together to do a specific task. function setup() sets up the drawing by running commands once, and function draw() draws the sketch by running commands over and over.

As you write code, it's helpful to know rules about syntax. Syntax, pronounced SIN-tax, means the arrangement of letters and symbols. If you use correct syntax, the computer will understand what you want to do. If you use incorrect syntax, the computer will show an error message and stop drawing.

Here are some important syntax rules:

  • Most commands end with ;
    which is called a semicolon.
  • You can include extra space before or after commands or numbers. For example, both
    background ( 51, 100, 100 ); and
      background(51,100,100); are good.
  • You can include extra lines between commands (press enter or return).
  • In function setup() and function draw() you can include as many commands as you like, but they all must be written between { and }. You can think of { } like bookends, with all the commands like books lined up between them.
  • The order of commands is very important. You must write function setup() and its commands before function draw(). You must write commands like fill and stroke before the shape that they color. The computer reads commands from top to bottom one at a time and will do what you say exactly, so it's your responsibility to put commands in a sensible order.

9. More setup() and draw()

Tinker

Click ▶Play. Just one ellipse bounces around the screen. What happened?
The background moved from function setup() to function draw(). That changes everything! Now...
• background → paints yellow background
• ellipse → draws ellipse at random position
• background → paints yellow background again, erasing previous ellipse
• ellipse → draws ellipse at a new random position
... and so on!

  • To make the first animation, change the background command to include a random hue.
  • To make the second animation, change the ellipse command to include random width (but nothing else should be random).
