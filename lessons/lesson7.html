2. Crossing the Screen

Tinker

Click ▶Play. You'll see one ellipse traveling across the screen from left to right. The code almost the same as in the previous step, but there's one difference. Can you find it?
Try these challenges: Cut the background command out of function draw() and paste it into function setup() instead. Then click ▶Play. What happens? Cut the background command out of function setup() and paste it back into function draw(). Then click ▶Play. What happens?

Explanation

Here's why the position of the background command matters. When the background command is in function draw(), the background color is painted on the screen over and over, once each time that function draw() runs. Each time the background is painted, it obscures any previously-drawn ellipse. So you see just one ellipse at a time. When the background command is in function setup(), the background is painted just once, when the project begins. Meahwhile, the ellipses in function draw() are painted over and over, each time draw() runs, so you see a line of them.

3. Transparent Backgrounds

Tinker

Click ▶Play. You'll see a moving ellipse again, but it has a tail.
Try these challenges: Notice that we've changed the background command by adding a fourth argument, 0.2. Change that argument to other decimals between 0 and 1 — for example, try 0.2 to 0.1 or 0.05 or 0.6
Which numbers make the longest tails? The shortest tails? Make the fourth argument to 0. What happens? Make the fourth argument to 1. What happens?

Explanation

The fourth argument to the background command changes its transparency — how much you can see through it. Here's how transparency works. You can make backgrounds with many different levels of transparency, from not-at-all transparent to completely transparent. Transparency is a sliding scale. The greater the transparency, the more previously-drawn shapes you'll see. For example, a fourth argument of 0.2 makes the background mostly transparent. You can see several previously-drawn ellipses, which appear as a tail. Confusingly, the smaller you make the fourth argument, the greater the transparency. So changing the fourth argument from 0.2 to 0.1 makes the background more transparent. You see a longer tail of previously-drawn ellipses. When the transparency is 0, the background is completely transparent, and you see all previously-drawn ellipses.
