1. mouseX and mouseY

Tinker

Click ▶Play. Then move the mouse around the canvas. You should see a white ellipse that follows the mouse. Can you figure out which part of the code tells the ellipse to follow the mouse?
Try these challenges: Delete mouseX and type a number instead. Click Play. What happens? Click Revert to restore the previous script. Add a 2nd ellipse with mouseY as the first argument and mouseX as the second argument. Click ▶Play. What do you see? Why?

Explanation

Remember that in the ellipse command, the first argument tells the ellipse its distance from the left edge, also called its x-coordinate. The second argument tells the ellipse its distance from the top edge, also called its y-coordinate. If you tell ellipse to go to mouseX, mouseY it will go wherever the mouse is!

mouseX is the distance from the mouse pointer to the left edge, also called the mouse's x-coordinate. mouseY is the distance from the mouse pointer to the top edge, also called the mouse's y-coordinate.

Shapes like ellipses have coordinates that describe their positions on the canvas, and the mouse pointer has coordinates too!

2. mouseX Again

Tinker

mouseX is called a variable. A variable is a label — in this case, mouseX — that stands in for a value like a number. Usually that value changes, or varies. For example, label or variable called mouseX stands in for the mouse's distance from the left edge of the canvas, which changes every time you move the mouse. When the computer sees the label mouseX, it calculates the mouse's distance from the left edge and plugs in that number to use instead of the label.
Try these challenges: In the window below, move your mouse sideways from left to right. What happens to mouseX? Now move the mouse sideways from right to left. What happens to mouseX? Now move the mouse straight up and down. What happens to mouseX now?

If you move the mouse from left to right, mouseX increases in value.

If you move the mouse from right to left, mouseX decreases in value.

If you move the mouse straight up and down, mouseX doesn't change.

3. mouseY Again

Tinker

mouseY is also a variable. Its value is the mouse's distance from the top edge of the canvas.
In the window below, move your mouse from top to bottom. What happens to mouseY? Now move the mouse from bottom to top. What happens to mouseY? Now move the mouse straight sideways. What happens to mouseY?

If you move the mouse from top to bottom, mouseY increases in value.

If you move the mouse from bottom to top, mouseY decreases in value.

If you move the mouse straight sideways, mouseY doesn't change.

5. Twins

Tinker

Click ▶Play. Move your mouse through the window below. Notice that there are 2 ellipses. Can you figure out how the code makes the second ellipse follow the first? Try these challenges. Add a third ellipse that moves sideways with the mouse but appears below the others. Add a fourth ellipse that moves sideways with the mouse but appears above the others. You'll need to subtract! To subtract, use – instead of +.

Explanation

In the previous two tinkers we've explored the idea that mouseX and mouseY stand in for distances. The distances are, of course, numbers. You can use mouseX and mouseY anywhere the code expects a number, including in calculations. Notice that the second ellipse calculates its position by adding mouseX+100, making that ellipse appear 100 pixels to the right of the first ellipse and the mouse.

To make a third ellipse that moves sideways with the mouse but appears below the others, we can add a small number to the mouse's y-coordinate, like this:
ellipse(mouseX, mouseY+50, 10, 10);

To make a fourth ellipse that moves sideways with the mouse but appears above the others, we can subtract a small number from the mouse's y-coordinate, like this:
ellipse(mouseX, mouseY-20, 10, 10);

It's OK if your code is a little different from these examples as long as two new ellipses appear above and below the originals.

6. Reflections

Tinker

Move your mouse through the window below. The two dots reflect each other's sideways movement, moving in opposite directions by the same amount. In a moment, we'll write code to create this animation, but first it's important to explore its logic.

Explanation

The green dot moves sideways with the mouse. Its x-coordinate is mouseX.
The blue moves sideways in the opposite direction. Its x-coordinate is 250-mouseX. The number 250 is the right edge of this particular canvas.
How do these settings create opposite movement? We know that mouseX measures the distance from the mouse pointer to the left edge. The calculation 250-mouseX subtracts that same distance from the right edge. So the two dots are always equal distances away from their opposite edges, reflecting each other's sideways movement.

7. Reflections Again

Tinker

Click ▶Play. Move your mouse through the window below. Notice that there are 2 ellipses. As the first ellipse moves right, the second ellipse moves left by the same amount. As the first ellipse moves down, the second ellipse moves down by the same amount. Why?
Try these challenges: Add a third ellipse that moves sideways with the first ellipse, but moves down when that ellipse moves up. There seems to be room for a fourth ellipse! Can you figure out how to add it?

The code below creates a third ellipse that moves sideways with the first ellipse but moves down it moves up.
ellipse(mouseX, 200-mouseY, 10, 10);
The number 200 is the height of the canvas.

The code below creates a fourth ellipse that moves sideways with the second ellipse but moves down it moves up.
ellipse(250-mouseX, 200-mouseY, 10, 10);

Here is the syntax of the line command:

To make the line start at the top right corner, you can use this command:
line(250, 0, mouseX, mouseY);
where 250, 0 is the top right corner of the canvas.

To change the thickness of the line and give it a random color, add these two commands to the script above line:

strokeWeight(4);
stroke(random(360), 100, 100);

9. pmouseX and pmouseY

Tinker

Click ▶Play. Move your mouse through the window below. You'll draw a line as if the mouse were a pen.
How does it work? The new variables pmouseX and pmouseY contain the previous mouse x- and y-positions. They make the line command draw a tiny line segment from the position the mouse was at an instant earlier to the position the mouse is at now. As you move the mouse, those tiny line segments knit together into a continuous line that feels like a drawing pen.
Try these challenges: Make the line color random. Make the line thickness (strokeWeight) random.
